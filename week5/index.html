    Testing and Debugging

    Alert

    The cool thing with alert() is that it stops a program from running until OK is clicked. That means we can use it as a breakpoint.

    Console

    Writing a few console.log() inside an if statement for example can be a good way to see what values are inside of certain variables

    Debugger

    Inside the web debugging tool (inspect), you can actually hover over a variable to check its value.

    Exceptions

    A finally block can be added after a catch block. This will always be executed after the try or catch block, regardless of whether an exception occurred or not. It is useful if you want some code to run in both cases.

    Test-driven Development (TDD)

    1. Write tests (that initially fail)
    2. Write code to pass the tests
    3. Refactor the code
    4. Test refactored code
    5. Write more tests for new features

    Jest

    Jest is a TDD framework that allows you test files and functions very easily.

    You can install it with the following command:

    npm install -g jest

    To actually test the file you'd like to test, simply navigate to the folder that contains that file and enter the following command:

    jest -c {}

    Project Management

    Trello

    Trello divides work into teams, then into boards(projects). Boards are divided into tasks and tasks can be divided into smaller tasks inside of a checklist.

    The Teamgantt power-up allows you to add Gantt charts to your project management.

    Advice from a teacher: Don't let tasks look empty. Try to fill them up with as many details as you can so that the members of the teams know exactly what they have to do. It is also useful for you because you can know how far team members are in each task.
